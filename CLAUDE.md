# ズレ計 — プロジェクト仕様書

政府・公的機関の「見通し」と「実績」を並べて記録し続ける静的サイト。
断罪も提言もしない。予測値・実績値・出典を毎年並べるだけの「計器」。

## 思想(変更しないこと)

- **思想は正直に、運用は無色に。** 動機はAboutページに全部書く。データページには形容詞を一切置かない。
- 政府批判サイトにしない。「楽観的すぎる」「ひどい」等の評価語はデータページ禁止。数字と出典リンクのみ。
- すべての数値に一次資料への出典リンクを付ける。出典のない数字は掲載しない。
- 目標は、公的な独立検証機関(日本版OBR/CBO)ができてこのサイトが不要になること。

## 情報公開の3段階

このプロジェクトの文書は、公開範囲に応じて3段階に分けて管理する。

1. **検討過程**: 設計や表現の迷い・比較検討そのもの。コミットしない(会話・作業メモのみ)。
2. **思想・動機**(多くの人に読んでほしい): なぜこのサイトを作るか。`about.md`に集約し、サイトのAboutページの原稿とする。
3. **実装・データの詳細**(監査・貢献者向け): フェーズ計画・データ設計・技術構成など、このCLAUDE.md本体や`data/README.md`のような形式仕様。一般読者向けの文章ではないが、検証可能性のために公開する。

## フェーズ計画

### Phase 1: 政府経済見通し(実質・名目GDP成長率)

- 内閣府「経済見通しと経済財政運営の基本的態度」の当初見通し(毎年12月頃閣議了解/1月閣議決定)の成長率予測値を収集
- 対象期間: 可能な限り遡る(最低1980年度〜)
- 実績値: 内閣府「国民経済計算」の確報値
- 過去の見通しアーカイブ: https://www5.cao.go.jp/keizai1/mitoshi/mitoshikako.html
- 先行研究(検証の参考、数値の妥当性チェック用):
  - キヤノングローバル戦略研究所(2018): 名目成長率、1998-2017年度で的中5/20回
  - ニッセイ基礎研究所(2019): 実質成長率の予測誤差検証

### Phase 2: 税収見積もり

- 財務省の一般会計税収の当初予算見積もり vs 決算額
- 補正予算での修正履歴も記録できると望ましい

### Phase 3: 出生率・人口推計

- 社人研「日本の将来推計人口」の合計特殊出生率仮定(中位) vs 実績
- 歴代推計(1992, 1997, 2002, 2006, 2012, 2017, 2023...)を重ねた「扇形」グラフが目玉

## データ設計

- DBなし。リポジトリ内のCSVをソースオブトゥルースとする
- 形式例: `data/gdp_forecast.csv`
  - columns: fiscal_year, forecast_real, forecast_nominal, actual_real, actual_nominal, forecast_source_url, forecast_published_date, actual_source_url, notes
- 実績値は改定されるため、取得日と確報/速報の別をnotesに記録
- 更新は年1回の手動運用を前提とする(自動化しない。壊れない仕組み優先)
- `csv.js`のパーサはクォート付きフィールド非対応の単純split(",")。自由記述欄(notes, reason等)に読点以外のカンマを含めない。カンマが必要な場面は「、」で代用する

## 技術構成

- 静的サイト。Cloudflare Pages(プロジェクト名 `zurekei-site`)
- ドメイン: zurekei.org をCloudflare Registrarで取得(取得〜DNS〜Pagesまで全部Cloudflareで完結)
- **デプロイはGitHub連携ではなくwranglerからの直接アップロード**。GitHubへのpushではデプロイされない。反映するには必ず下記を実行する
  ```
  npx wrangler pages deploy . --project-name=zurekei-site
  ```
  (Deployments画面にブランチ名とコミットメッセージが出るのはwranglerがgitメタデータを付けているだけで、Git連携ではない)
- アクセス解析: Cloudflare Web Analytics。Pagesプロジェクトの Metrics タブで有効化済みで、ビーコンは配信時に全HTMLへ自動挿入される。**有効化・設定変更は次回デプロイから反映される**ためデプロイを1回挟む
- フレームワーク: シンプル優先。Vite + React または素のHTML+JS。ビルドが複雑になるなら素のHTMLでよい
- グラフ: インタラクティブ必須(下記)

## デザイン方針

- コンセプトは「計器」。参照イメージは気象庁・日銀統計のような「そっけないのに信頼できる」佇まい。装飾やイラストではなく、数字の組版・余白・グリッドの精度で信頼を作る
- 数字が主役。等幅系で数字が美しいフォントを選ぶ(tabular numsを効かせる)
- **開幕10秒の演出(サイトの目玉)**: トップページを開くと、実績の線がすっと引かれ、歴代の予測線が1本ずつ重なり、ズレが扇形に開いていく。文章ゼロで主張が伝わるアニメーション。OGP画像はこの完成形のグラフ(SNSで流れてきた人に絵で刺さる)
- prefers-reduced-motion では静的表示にフォールバック
- 内容が地味なぶん、見せ方は動的に:
  - 年を選ぶスライダー/セレクタで「その年に政府が見ていた未来」の予測線がアニメーションで描かれ、実績線と重なる
  - 予測vs実績の乖離(誤差)が選択に応じて動的に表示される
  - Phase 3の出生率は歴代推計を1本ずつ重ねていく「扇が開く」演出が目玉
- 評価語なし。色で善悪を示唆しない(赤=悪、のような色使いは避け、予測/実績の区別のみに色を使う)
- モバイルファースト(SNSシェアからの流入想定)
- OGP画像はグラフそのもの(シェアされたときに絵で伝わる)

## 経緯メモ(なぜこのプロジェクトか、一行で)

政策のズレの上流には「予測が外れても誰も記録しない=学習ループの不在」がある。独立検証機関(日本版OBR)ができるまでの民間先行版として、権限ゼロでできる「予測と実績のズレの継続記録」を一市民が作る。
