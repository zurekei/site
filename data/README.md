# data/ — データ形式仕様書

`ズレ計`が扱う全データの列定義・記法規約をまとめる。監査者・貢献者向けの技術文書であり、一般読者向けの文章ではない(公開範囲の位置づけは`../CLAUDE.md`の「情報公開の3段階」を参照)。

## 1. 方針

- このディレクトリ内のCSVがソースオブトゥルースである。DBは持たない。
- 出典のない数字は掲載しない。すべての数値行に一次資料へのURL(`*_source_url`列、`fertility_actual.csv`では`source_url`列)を付す。
- 実績値は後から改定されることがある(例: 国民経済計算の確報、国際収支の速報→確定)。そのため取得時点や確報/速報/概数の別を`notes`列に記録する。
- 更新は年1回の手動運用を前提とする(自動化しない)。

## 2. 用語

- **年度(fiscal_year)**: 日本の会計年度。4月開始・翌年3月終了で、開始年の西暦で表記する(例: `2024`は2024年4月〜2025年3月)。`gdp_forecast.csv`など大半のファイルはこの列を使う。
- **暦年(year)**: 1月開始・12月終了の西暦。`fertility_actual.csv`・`fertility_forecast.csv`の`target_year`はこちらで、年度ではない。
- **当初見通し/当初予算**: 対象の年度が始まる前(経済見通しは前年12月頃の閣議了解・1月頃の閣議決定、予算は前年12月〜1月頃の閣議決定)に示された最初の予測値。年度途中の補正予算・上方/下方修正は含まない。
- **実績/決算**: 年度終了後に確定した値。指標によって呼び方が異なる: 国民経済計算(GDP)は「確報」、消費者物価・失業率は統計調査の確定値、税収・国債発行は予算に対する「決算」額。
- **速報/概数/確報**: 統計の確定段階を表す語。速報・概数は将来改定される可能性がある暫定値、確報は改定が想定されない確定値を指す。指標ごとに三段階すべてが存在するとは限らない。
- **vintage(版)**: `fertility_forecast.csv`のみで使う語。社人研「日本の将来推計人口」が公表された年(=推計の版)を指す。1つのvintageの中に、複数の`target_year`(推計対象の将来暦年)についての仮定値が並ぶ。

## 4. 共通規約

### 4.1 列構成パターン

`bond_issuance_forecast.csv`・`cpi_forecast.csv`・`current_account_forecast.csv`・`jgb_total_issuance_forecast.csv`・`tax_revenue_forecast.csv`・`unemployment_forecast.csv`の6ファイルは、以下の共通パターンに従う。

```
fiscal_year, forecast_<指標>, actual_<指標>, forecast_source_url, actual_source_url, notes
```

このパターンに従わないファイルとその理由は以下の通り。

- `gdp_forecast.csv`: 実質・名目の2指標を1ファイルに収めているため9列になる。加えて`forecast_published_date`列を持つ点、`actual_source_url`が`forecast_published_date`の後に来る点で、上記の列順とは異なる。`actual_source_url`は実質・名目のURLを`ラベル:URL`形式で1セルにまとめており、これは同一ファイルの他5ファイルには見られない記法である。
- `fertility_forecast.csv`: `fiscal_year`ではなく`vintage_year`/`target_year`を使う。1回の公表(vintage)が複数の対象年の仮定値をまとめて含むため、単年の見通し/実績ペアという枠組みに合わない。`actual_*`列を持たず、実績は`fertility_actual.csv`に分離されている。
- `fertility_actual.csv`・`corrections.csv`: それぞれ目的が異なる最小構成のファイルであり、上記パターンの対象外。

### 4.2 空欄と0の違い

- **空欄**: 値が存在しないのではなく、「まだ記録できていない」ことを表す。理由は主に3つ — (a)当該年度がまだ終了しておらず実績が未確定、(b)一次資料を確認できておらず見通し値が未収集、(c)原資料に該当項目の記載がない。理由は`notes`に記す。
- **`0`**: 出典において明確に0(円・%など)と確認できた場合にのみ記録する。例: `bond_issuance_forecast.csv`の1947〜1964年度は、財務省資料の脚注に「収支均衡予算のため国債発行なし」との明記があるため`0`を記録している。単に値が小さい・未確認という理由で`0`を記録してはならない。
- 描画側(`csv.js`の`toNum()`)は空文字列を`null`に変換する。`null`は「データ未収集」等として扱われ、グラフの折れ線は値が存在しない年をまたいで補間されない(隣接しない年同士を直線で結ばない)。

### 4.3 notesの記法

`notes`は複数の断片を`" / "`(半角スペース+スラッシュ+半角スペース)で連結する自由記述欄である。断片の先頭に角括弧のタグを付ける慣習があり、現状使われているタグは次の6種。

| タグ | 用途 |
|---|---|
| `[出典]` | 数値の出典・確認方法の説明 |
| `[見通し原文]` | 見通し文書の該当箇所を原文のまま引用したもの。数値自体は`forecast_*`列と重複するが、検証用に残している |
| `[見通し]` | 転記方法・注意点など、見通し値の取得に関する補足 |
| `[見通しのみ]` | 実績値がまだ存在しない(年度未終了・未確定)ことの明示 |
| `[実績]` | 実績値の出典・注記、または当該年度の背景事象の記述(例:「リーマンショック年度。」) |
| `[参考]` | 補正後予算額など、`forecast_*`/`actual_*`列に収録していない付随的な数値 |

例(`gdp_forecast.csv` 2008年度): `[実績]リーマンショック年度。 / [見通し原文]こうした結果、平成20年度の国内総生産の実質成長率は、2.0％程度（名目成長率は2.1％程度）になるなど、別添の主要経済指標のとおりと見通される。`

これらのタグは表示側で機械的に解釈される。`chart.js`の`extractEventNote()`は`" / "`で分割した断片のうち`[実績]`で始まるものだけを抽出し、グラフのイベント注記として表示する。それ以外のタグ(タイポや未知のタグを含む)は読み捨てられ画面には出ない。したがって、当該年度の出来事をグラフ上に注記したい場合は`[実績]`タグを使う必要があり、他のタグで書いても表示されない。

### 4.4 半角カンマ禁止

`csv.js`のパーサはクォート付きフィールドに対応しない単純な`split(",")`である(`csv.js` 3行目`headers = lines[0].split(",")`、5行目`cells = line.split(",")`)。そのため自由記述欄(`notes`など)に半角カンマ(`,`)を含めると、その行の列がずれて壊れる。カンマが必要な場面は読点(`、`)で代用する。全角カンマ(`，`)も紛らわしいため使わない。

行を追加・編集した際は、その行のフィールド数(カンマの個数+1)がヘッダ行のフィールド数と一致することを確認する。これは半角カンマが自由記述欄に紛れ込んでいないかの簡易的な検査になる。

### 4.5 丸め・単位換算

- 原資料が億円単位の場合は兆円に換算して記録する(`bond_issuance_forecast.csv`・`tax_revenue_forecast.csv`・`current_account_forecast.csv`)。単純換算(億円÷10000)の場合は小数4桁まで残ることがある(例: `bond_issuance_forecast.csv`の`28.6471`)。原資料自体が既に兆円単位に四捨五入して掲載している場合(`current_account_forecast.csv`)は、その旨を`notes`に「四捨五入」等と明記する。
- %表示の指標(`gdp_forecast.csv`・`cpi_forecast.csv`・`unemployment_forecast.csv`)は原資料の精度(小数1桁)をそのまま記録する。
- 出生率(`fertility_forecast.csv`・`fertility_actual.csv`)は比率であり%表記ではない。小数桁数は原資料のまま統一していない(§3の該当ファイル参照)。
- 数値は半角ハイフン`-`を負号として使う(全角マイナス`−`や`▲`は使わない)。

## 5. 更新頻度と訂正方針

年1回の手動更新を前提とする(自動化はしない)。指標ごとの目安は以下の通り。

- **政府経済見通し(当初値)**: GDP成長率・CPI・失業率・経常収支の当初見通しは、例年12月頃の閣議了解・1月頃の閣議決定に合わせて12〜1月に追記する。
- **決算・確報**: GDP・CPI・失業率の実績、税収決算、国債発行実績は、例年7〜12月頃に確定情報を追記する。
- **人口推計**: 出生率の仮定値・実績は、社人研の公表(不定期。過去は1992, 1997, 2002, 2006, 2012, 2017, 2023年)に合わせて追記する。

訂正方針: 一度公開した数値の誤りは、値を黙って上書きしない。訂正する場合は`data/corrections.csv`に1行追加して履歴を残し、`corrections.html`で一覧公開する。`before`/`after`列に訂正前後の値、`reason`列に訂正理由を記録する。

## 6. 検証方法

各行の`forecast_source_url`/`actual_source_url`(`fertility_actual.csv`では`source_url`)は一次資料への直接リンクである。監査者は該当URLを開き、その行の`fiscal_year`(または`year`・`vintage_year`+`target_year`)に対応する数値がCSVの値と一致するかを突き合わせて確認できる。

注意点:

- 財務省の一部資料(例: `sy202604f.pdf`)は同一URLのままファイル内容が更新される「時点版」の資料である。この場合`notes`に取得時点を「(ファイル名、令和X年Y月時点版)」の形式で明記しているので、URLへアクセスした時点の最新版と記録時点の版が異なる可能性がある点に留意する。
- 公式サイトのURLが後年別内容に差し替えられている場合、Wayback Machineのアーカイブ版URL(元のURLをそのまま内包する形式)を出典として用いることがある。`jgb_total_issuance_forecast.csv`の1990〜2016年度の国債発行計画(`forecast_source_url`)がこれに該当する。
- `gdp_forecast.csv`の`actual_source_url`のように1セルに複数のURLがまとめられている列は、`ラベル:URL`の形式でラベル(`実質`/`名目`)ごとに分離できる。
- `notes`の`[実績]`タグにはグラフに表示される背景事象の記述が入ることがあるが、この記述自体は評価語を含まない事実の記録である(例:「バブル期のピーク。」「消費税率引き上げ・アジア通貨危機の年度。」)。
